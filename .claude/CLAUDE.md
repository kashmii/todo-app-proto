# テストコード作成時の注意事項

## テストコードの品質
- テストコードは実際の機能を検証すること
- ｀expect(true).toBe(true)｀のような無意味なテストは避けること
- モックは最小限にとどめ、実際の動作を確認できるようにすること

## ハードコーディングの禁止
- テストを通すためだけのハードコーディングは禁止
- テスト用のマジックナンバーを本番コードに持ち込まないこと
- 環境変数や設定ファイルを使用して、テスト環境と本番環境を分けること

## 実装前の確認
- 機能の使用を正しく理解してからテストコードを書くこと
- 不明点があればカリのコードを書くのではなくユーザーに確認すること
